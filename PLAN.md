# Graphwar 游戏开发计划

## 游戏规则

### 基本概念
- 双方各拥有 **5 个方格**（矩形区域，每个 1.2×1.0 单位），在 y 轴两侧**各自随机分布**
- 方格同时作为 **己方发射点** 和 **对方攻击目标**
- 每一回合，系统**自动随机选择**当前玩家一个未被摧毁的方格作为发射点
- 玩家输入数学函数 y = f(x)，系统自动添加常数 C，使图像经过发射点
- 函数图像从发射点开始向对方方向延伸
- 若图像进入对方任意一个方格，则命中，该方格被摧毁
- 函数由**消耗点数系统**约束：每个运算符、常量、数学函数、变量都消耗一定点数
- 每轮获得一定点数，且随轮数递增
- **无时间限制**，**无输入遮挡**（双方可见）

### 胜负条件
- 先摧毁对方全部 5 个方格的一方获胜
- 每一局结束后可选择再来一局

### 开始界面
- 游戏启动时显示开始界面，包含标题、简介和"START GAME"按钮
- 点击后进入游戏界面
- 游戏结束后显示"Play Again"按钮可重新开始

### 方格布局（随机生成）
- 每局开始前，双方各自的 5 个方格在己方半区**随机生成**
- 玩家 1（左侧）：x ∈ [-12, -3]，y ∈ [-7, 7]，方格之间最小间距 1.5 单位（防止重叠）
- 玩家 2（右侧）：x ∈ [3, 12]，y ∈ [-7, 7]，方格之间最小间距 1.5 单位
- 每个方格大小 **0.6×0.6** 单位
- 方格以半透明色块绘制，被摧毁后变灰或消失
- **一轮 = 双方各射击一次**，轮数在双方都射击后才增长

### 资源点数系统

**每轮可用点数（随轮数递增，一轮 = 双方各射击一次）：**
- 第 1 轮：4 点
- 第 2 轮：5 点
- 第 3 轮：6 点
- 第 N 轮：3 + N 点

**各语法单元消耗点数：**

| 语法单元 | 消耗 | 示例 |
|---------|------|------|
| 变量 `x` | 1 | `x` |
| 整数常量 | 1 | `2`, `-5` |
| 小数常量 | 每数字 1 | `3.14` → 4 |
| `+`, `-`（二元） | 1 | `x+2` |
| `*` | 2 | `x*3` |
| `/` | 2 | `x/2` |
| `^`（乘方） | 3 | `x^2` |
| `sin`, `cos`, `tan` | 3 | `sin(x)` |
| `sqrt`, `abs` | 2 | `sqrt(x)` |
| `log`, `ln`, `exp` | 3 | `log(x)` |
| `asin`, `acos`, `atan` | 4 | `asin(x)` |

**计费规则：**
- 解析用户输入的函数表达式 → 遍历 AST 节点 → 累加各节点消耗
- 若总消耗 > 当前轮可用点数，拒绝发射并提示"点数不足，剩余 X 点"
- 若总消耗 ≤ 可用点数，扣除对应点数并执行发射

**策略意义：**
- 早期轮次点数少，倒逼玩家使用简单的线性/二次函数，考验对基础函数轨迹的预判
- 后期轮次点数多，允许构造复杂的复合函数绕过障碍，增加翻盘可能性
- 玩家需在函数复杂度与命中率之间做权衡

```
Graphwar/

│   │   ├── Player.h/cpp                  # 玩家数据（id、颜色）
│   │   ├── Square.h/cpp                  # 方格数据（坐标、大小、存活状态）

│   │   └── GameViewModel.h/cpp           # ViewModel：管理游戏逻辑、暴露属性/信号/槽
│   ├── view/
│   │   ├── MainWindow.h/cpp              # 主窗口（含开始界面 QStackedWidget）
│   │   ├── GameCanvas.h/cpp              # 画布：坐标系、方格、函数曲线
│   │   └── FunctionInput.h/cpp           # 函数输入面板
│   ├── parser/
│   │   ├── Token.h                       # Token 类型定义
│   │   ├── Tokenizer.h/cpp               # 词法分析器
│   │   ├── Expression.h/cpp              # AST 节点定义
│   │   ├── Parser.h/cpp                  # 递归下降解析器
│   │   └── Evaluator.h/cpp               # 表达式求值器
│   └── utils/
│       └── Geometry.h                    # 坐标变换、碰撞检测工具
```

**MVVM 职责划分：**

| 层 | 职责 | 文件 |
|----|------|------|
| **Model** | 纯数据容器，无业务逻辑，可被序列化 | `GameModel`, `Player`, `Square` |
| **ViewModel** | 持有 Model，封装全部游戏逻辑，通过 Q_PROPERTY / signal 暴露状态，通过 slot 接收用户操作 | `GameViewModel` |
| **View** | 被动渲染，通过信号绑定 ViewModel 的属性变化，用户操作调用 ViewModel 的槽函数 | `MainWindow`, `GameCanvas`, `FunctionInput` |

## 核心模块详细设计

### 1. 数学表达式解析器

**支持的语法：**
- 运算符：`+`, `-`, `*`, `/`, `^`（乘方）
- 数学函数：`sin`, `cos`, `tan`, `asin`, `acos`, `atan`, `sqrt`, `abs`, `log`, `ln`, `exp`
- 注意：`log` 和 `ln` 的**真数自动取绝对值**，避免负数导致无定义
- 常数：支持整数和小数（如 `2`, `-3.14`）
- 变量：`x`
- 输入示例：`2*x^3 + sin(x) + 5`

**流程：**
```
输入字符串 → 词法分析(Token列表) → 递归下降解析 → AST → 求值/计费
```

**自动常数调整：**
- 由于函数必须经过发射方格中心 (cx, cy)，系统自动计算常数项
- 用户输入函数 f(x) → 系统计算 C = cy - f(cx) → 实际绘制函数为 f(x) + C
- 此时有 f(cx) + C = cy，保证图像必过发射点
- 用户无需手动调整常数，直接在输入框输入表达式即可

### 2. 游戏引擎 (GameViewModel)

**状态机：**
```
WAITING_INPUT → ANIMATING → BOUNDS_CHECK → ROUND_END / GAME_OVER
```

**详细流程：**
1. 轮到当前玩家 → 系统自动随机选择当前玩家一个未摧毁方格作为发射点，更新可用点数显示
2. 玩家输入函数 → 点击发射
3. **点数校验**：解析 AST 计算函数总消耗，若超过当前轮可用点数则拒绝并提示
4. 系统自动计算常数 C = cy - f(cx)，调整函数图像使其经过发射方格中心
5. 进入动画阶段：从发射点向**对方方向**逐帧延伸轨迹
6. 碰撞检测：每帧检查轨迹点是否落在对方任意未摧毁方格内（可穿透多个）
7. 若命中：标记对方所有被经过的方格为已摧毁，**动画不停止**，继续延伸
8. 若轨迹超出边界（|x| > 22 或 |y| > 14）仍未命中任何格子，Miss!
9. 若轨迹超出边界但命中过格子，Hit! 显示命中数量
10. 检查胜负：若一方 5 个方格全被摧毁，游戏结束（中途也可直接结束）

### 3. 画布渲染 (GameCanvas / View)

**与 ViewModel 的绑定方式：**
- GameCanvas 通过信号连接 GameViewModel 的属性变化
- ViewModel 发射 `trajectoryUpdated` 信号时，Canvas 触发重绘
- ViewModel 发射 `phaseChanged` 信号时，Canvas 更新提示文字
- Canvas 不直接修改任何游戏数据，仅作绘制

**坐标系：**
- 世界坐标范围：x ∈ [-14, 14], y ∈ [-10, 10]
- 像素尺寸约 900×700，自适应窗口大小

**绘制内容（按 z-order）：**
1. 背景（深色）
2. 坐标轴（灰色实线）
3. 双方的方格（半透明矩形，不同颜色，被摧毁的标记为灰色）
4. 当前动画轨迹（亮色，逐渐延伸），仅向对方方向延伸（**不保留**之前的轨迹）
6. 轨迹上的"弹头"（高亮圆点，沿曲线运动）
7. 选中方格的边框高亮（当前自动选中的发射方格）
8. 回合提示文字

**动画机制：**
- QTimer 驱动，帧间隔 16ms（~60fps）
- 每帧向对方方向扩展 **0.15** 单位步长（约 2 秒横跨全场）
- P1（左侧）→ 向右延伸，P2（右侧）→ 向左延伸
- 图像穿过对方格子后**不停止**，继续向边界延伸

### 4. 碰撞检测

- 每帧对动画新绘制的轨迹点进行检测
- 对对方每个未摧毁方格：检测点是否在矩形区域内
- 区域判定：`cx - w/2 ≤ px ≤ cx + w/2 && cy - h/2 ≤ py ≤ cy + h/2`（与方格可视大小一致）
- 命中后立即停止动画，标记结果

### 5. UI 布局

**流程：MainWindow 使用 QStackedWidget 切换两页**

**第 0 页 — 开始界面：**
```
┌──────────────────────────────────────────────────┐
│                                                  │
│                                                  │
│                 GRAPH WAR                         │
│      Two players, one curve.                     │
│    Fire functions at each other!                 │
│                                                  │
│              [ START GAME ]                      │
│                                                  │
│                                                  │
└──────────────────────────────────────────────────┘
```

**第 1 页 — 游戏界面：**
```
┌──────────────────────────────────────────────────┐
│  Player 1      Round 3    Pts:8      Player 2   │  ← 顶部状态栏
├──────────────────────────────────────────────────┤
│                                                  │
│              Game Canvas                         │
│    ┌───┐                        ┌───┐           │
│    │ P │    ←──── 曲线轨迹 ────→ │ P │           │
│    │ 1 │   ╱                     │ 2 │           │
│    │ ■  │  ╱                     │   │           │ ← 发射点
│    └───┘ ╱                      └───┘           │
│         ╱                                       │
│                                                  │
├──────────────────────────────────────────────────┤
│  f(x) = [________________]  消耗:3/8  [发射]    │  ← 输入区
│  [消息提示]                                      │
└──────────────────────────────────────────────────┘
```

**交互流程（MVVM 数据流）：**
1. 开始界面点击 START GAME → 切换到游戏页，ViewModel 初始化
2. 顶栏：显示当前玩家（高亮）、轮数、可用点数
3. 画布上：系统自动高亮选中的发射方格
4. 底部输入区：
   - 输入框输入表达式 → 实时计算消耗点数并显示 `消耗:N/总点`
   - 点击"发射" → 调用 `GameViewModel::launch(QString expr)`
   - ViewModel 内部：解析表达式 → 计算消耗 → 校验点数 → 计算常数 C → 进入动画阶段
5. 动画过程中：ViewModel 驱动 QTimer，每帧计算轨迹点 → 发射 `trajectoryUpdated` 信号 → Canvas 重绘
6. 碰撞检测在 ViewModel 内部完成（每帧检查）→ 命中时发射 `squareHit(int playerId, int squareIndex)` 信号
7. 动画结束后发射 `phaseChanged(WAITING_INPUT)` → 输入区解锁，自动切换回合
8. 游戏结束后显示 "Play Again" 按钮

## 方格随机生成算法

每局初始化时调用 `GameViewModel::generateSquares()`：

```
输入：玩家 ID，随机种子
输出：5 个不重叠的方格

算法：
1. 定义半区范围：
   - P1（左侧）：x ∈ [-12, -3]，y ∈ [-7, 7]
   - P2（右侧）：x ∈ [3, 12]，y ∈ [-7, 7]
2. 重复生成候选位置，直到产生 5 个方格：
   a. 在范围内随机选取 (cx, cy)
   b. 检查与已生成方格的最小间距 ≥ 1.5 单位（中心距）
   c. 通过则加入列表
3. 若尝试次数超过上限（如 200 次），用默认位置兜底
```

方格世界坐标 **0.6×0.6**（正方形），碰撞判定区域与方格大小一致。

**对称性说明：**
- 两侧半区各自独立随机，不做镜像对称
- 每局重启时重新随机，增加 replayability
