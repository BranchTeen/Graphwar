### 2.5 实际效果与现存问题

**实际效果**：

1. **代码修改精度高**：行级 Edit 工具能精确定位并替换代码片段，避免整文件重写
2. **跨文件重构能力**：将架构从 EventBus 迁移到 PropertyTrigger + Command 模式，涉及 20+ 个文件的逐文件修改，AI 能保持一致性
3. **多文件上下文理解**：AI 能同时理解 common/frame、viewmodel/commands、view/widgets 等不同层的代码，进行跨层修改
4. **视觉效果实现能力强**：AI 能够实现复杂的视觉效果，如函数轨迹发光效果、粒子爆炸特效、回合过渡动画等，通过多层绘制叠加和缓动函数实现出色的视觉体验
5. **文档同步更新**：在功能开发完成后，AI 能够自动同步更新 PLAN.md 文档，确保文档始终反映最新代码状态

**现存问题**：

1. **include 路径管理**：AI 在记住文件新位置（如从 model/ 移到 common/ 又移回 model/）方面偶尔会出现路径错误
2. **Qt 特有机制**：`QObject::connect` 在非 QObject 类中的调用需要显式加命名空间限定，AI 初期容易遗漏
3. **编译错误迭代**：AI 编写程序有小概率出现编译错误，需要人类反馈后修复
4. **UI 参数微调**：发光效果宽度、透明度、动画时长等参数需要人类多次调整才能达到满意效果，AI 缺乏直接的视觉感知能力

## 三、学习回顾与个人总结

### 3.1 知识掌握

#### 理论理解与概念应用

1. **MVVM 架构模式**：深入理解了 Model-View-ViewModel 的分层设计理念，实现了 View 层与 Model/ViewModel 的完全解耦
2. **状态管理**：通过 PropertyTrigger 实现了响应式状态更新，理解了单向数据流的设计思想
3. **命令模式**：使用 Command 模式封装用户操作，实现了操作的可撤销和可记录
4. **状态机设计**：游戏引擎采用状态机模式管理游戏阶段（WaitingInput → Animating → RoundEnd → Transition → GameOver）
5. **数学表达式解析**：实现了完整的表达式解析器（Tokenizer → ShuntingYard → Evaluator），理解了编译原理的基本概念

### 3.2 技能提升

#### 技术能力与实践水平的进步

1. **C++ 编程能力**：掌握了现代 C++ 特性（智能指针、Lambda 表达式、STL 容器），理解了内存管理和资源生命周期
2. **Qt 框架应用**：熟练使用 Qt6 的 Widgets、Multimedia、Network 模块，掌握了信号槽机制、自定义控件开发
3. **CMake 构建系统**：掌握了 CMake 的高级用法（条件编译、自定义目标、跨平台配置），实现了跨平台打包
4. **图形渲染**：掌握了 QPainter 的使用，实现了复杂的图形绘制（网格线、函数轨迹、发光效果、粒子系统）
5. **音频处理**：使用 Qt Multimedia 实现了 BGM 和音效播放，支持音量调节和静音
6. **序列化**：使用 JSON 实现了游戏存档系统，支持存档的保存、加载和删除

### 3.3 团队协作

#### 沟通、协调与合作经验

1. **需求拆解**：学会了将复杂需求拆解为可执行的小步骤，通过 TodoList 工具跟踪进度
2. **渐进式需求**：人类不需要一开始就提出完美需求，可以在迭代过程中逐步完善
3. **快速验证**：每次修改后及时编译验证，减少返工
4. **文档驱动**：维护一份与代码同步的规则文档，作为团队协作的基石
5. **反馈闭环**：建立了"需求 → 实现 → 验证 → 反馈 → 修复"的闭环流程

### 3.4 个人总结

#### 杨瑞潼

> 在本次项目中，我主要承担架构设计和技术选型的角色，从项目初期的 EventBus 模式到后来的 PropertyTrigger + Command 模式。项目后期，我负责了音频系统修复和回放系统架构设计，确保 BGM 和音效能够正确播放，支持音量调节和静音功能。最大的挑战是如何在跨平台环境下保证音频的稳定播放，解决 Qt Multimedia 在不同平台上的兼容性问题，同时还要确保音频系统与游戏逻辑的正确集成。通过引入 AudioManager 单例管理音频资源，并使用 QAudioSink 控制音频输出，我们成功实现了这一目标。

> 在与 AI 协作的过程中，我学会了如何将复杂的架构调整需求转化为 AI 能够理解和执行的具体任务。通过 TodoList 工具，我们将大规模重构拆解为多个小步骤，每完成一个步骤就编译验证，确保了重构过程的稳定性。每次添加新功能前，先更新 PLAN.md 的设计文档，让 AI 能够更好地理解需求并生成高质量的代码。
>
> 最终获得的成长是对"架构"在实际工程中意义的深刻理解。一个好的架构不仅是代码的组织方式，更是团队协作的基础。通过维护清晰的规则文档和分层结构，我们能够高效地与 AI 协作，在有限时间内完成了丰富的功能实现。同时，我也学会了如何在项目迭代过程中保持架构的一致性和可扩展性，这对我今后的软件开发工作将产生深远影响。

#### 李之彦

> 在本次项目中，我主要负责功能实现和效果调试。从核心游戏逻辑到各种 UI 效果（粒子特效、发光效果、过渡动画），我与 AI 紧密配合完成了所有功能的实现和优化。项目后期，我专注于视觉效果的提升，包括函数轨迹的发光效果、弹头的光晕效果、粒子爆炸特效以及回合过渡动画。最大的挑战是视觉效果的调试——发光效果的宽度、透明度，动画的时长、缓动函数等都需要多次调整才能达到满意的效果，而 AI 缺乏直接的视觉感知能力，这就需要人类的审美判断来指导优化方向。
>
> 在与 AI 协作的过程中，我发现"渐进式需求"是最有效的协作方式。不需要一开始就提出完美需求，可以先提出一个大致方向，然后根据实际效果逐步调整。例如，添加发光效果时，我先要求"添加函数轨迹绘制时的发光效果"，然后根据效果反馈"发光的效果再明显一点"、"当前轨迹改窄一点"，最终达到了满意的效果。这种交互式的协作方式让我能够充分发挥人类的审美优势，同时利用 AI 的代码实现能力。
>
> 最终获得的成长是对 C++ 和 Qt 框架的深入理解。通过阅读和修改大量代码，我对内存管理、信号槽机制、图形渲染等技术有了更深刻的认识。特别是在实现发光效果和粒子系统时，我学会了如何通过多层绘制叠加和透明度控制来创造出色的视觉效果。同时，我也学会了如何与 AI 高效协作，将 AI 的代码生成能力与人类的审美判断和架构思考相结合，实现了 1+1>2 的效果。这种人机协作模式让我对未来的软件开发方式有了新的思考。

## 结语

Graphwar 项目已完成全部功能开发，包括核心玩法、视觉效果、音频系统、存档系统、统计功能等。项目采用 MVVM + PropertyTrigger + Command 架构模式，实现了 View 层与业务逻辑的完全解耦，代码结构清晰，可维护性强。

在 AI 辅助下，我们高效地完成了功能开发和架构优化。AI 的代码生成能力大大提高了开发效率，而人类的架构思考和审美判断则确保了代码质量和用户体验。这种人机协作模式为软件开发带来了新的可能性，也让我们在学习过程中获得了独特的体验和成长。
